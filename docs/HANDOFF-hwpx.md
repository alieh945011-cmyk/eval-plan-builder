## 남은 작업 1: HWPX 생성 (핵심)

사용자 선택: **빈 양식 전체를 복제하고 3~6번을 채운 완성 문서** (폴백: 3~6번 표만 담은 별도 hwpx).

### 재료
- 빈 양식(학년별): `..\2026. 2학기 평가계획서 양식\2026학년도 2학기 {1,2,3}학년 교수학습 및 평가운영계획_(교과명).hwpx`
- 분석용 압축해제본(이전 세션 스크래치, 없으면 다시 풀 것): 본문은 `Contents/section0.xml` 단일 파일
- 표 구조 상세: 이 저장소 커밋 메시지와 아래 요약, 그리고 `src/preview.js`의 표 렌더 (HTML로 동일 구조 구현되어 있음 — 셀 병합 패턴의 참조 구현)

### 검증된 표 구조 요약 (hwpx 분석 완료 사항)
- hwpx = ZIP, 표는 `hp:tbl > hp:tr > hp:tc`, 병합은 `hp:cellSpan @colSpan/@rowSpan`, 텍스트는 `hp:t`
- 2022 양식(F1=2학년 빈양식): 3-가(성취기준별 표 반복) / 3-나(학기단위 A~E) / 4번 매트릭스 / 5번 성취율표 / 6번 영역별(수행과제 행 있음, 척도 A~E)
- 2015 양식(3학년): 3번 단일표(단원|성취기준|평가기준 상중하), 4번에 `전기학교반영여부` 행 추가, 6번에 수행과제 행 없음, 채점란 명칭 `채점기준`
- 빈 양식 셀에는 작성 지침 주석 텍스트가 들어 있음 → 채울 때 제거 대상
- 6번 하단 고정 3행: 기본점수/미참여/장기결석 + AI 유의사항 병합행

### 권장 구현 경로
1. `public/templates/`에 빈 양식 3종 복사 (파일명 단순화: `2022-g1.hwpx, 2022-g2.hwpx, 2015-g3.hwpx`)
2. `src/export-hwpx.js`: JSZip으로 로드 → section0.xml을 DOMParser로 파싱 →
   - 표 식별: 셀 텍스트 앵커로 탐색 (예: "평가 종류", "성취율", "평가 영역명")
   - 행 복제: 기준 행(`hp:tr`) cloneNode 후 `hp:t` 텍스트 치환. **셀 ID/페이지 분할 속성은 한글이 재계산하므로 대체로 안전하지만, 반드시 실파일을 한글에서 열어 검증**
   - 교과명 치환: 문서 전체에서 "(교과명)" → 과목명
3. 실패/시간 부족 시 폴백: kordoc `generate`(마크다운→HWPX) 또는 "3~6번 표만 담은 새 hwpx" — kordoc MCP가 이 PC에 설치되어 있음 (`npx -y kordoc`)
4. 검증: 생성 파일을 사용자가 한글 2020+에서 열기 (수동 왕복 — 체크리스트로 일괄 요청)

## 환경 주의사항 (이 PC)
- 로컬 `vite build`가 네이티브 크래시(0xC0000409)로 실패 — 원인 불명, 코드 문제 아님(커밋 상태도 실패, CI는 성공). 로컬 확인은 `npm run dev`로, 프로덕션 빌드는 push 후 GitHub Actions로.
- PowerShell 5.1 `Set-Content -Encoding UTF8`은 BOM을 넣어 package.json 등을 깨뜨림 — 파일 수정은 Write/Edit 도구로.
- 2015 hwp 원본의 페이지 경계 셀 융합·오타('증'→'중')는 `scripts/extract_2015.py`가 처리함. 데이터 재추출 시 그대로 실행하면 됨.
