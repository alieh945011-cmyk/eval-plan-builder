# 수행평가 평가계획 도우미

중학교 "교수학습 및 평가운영계획"의 3~6번(성취기준·반영비율·수행평가 세부기준)을 설계하고
표 복사·엑셀로 내보내는 교사용 무료 도구. 2015 개정(3학년, 상·중·하)과 2022 개정(1·2학년,
A~E + 학기 단위 성취수준)을 모두 지원한다.

- 접속: https://alieh945011-cmyk.github.io/eval-plan-builder/
- 데이터 검수(내부용): 주소 뒤에 `#review`

## 화면 구성

| 주소 | 화면 |
|---|---|
| `#` (기본) | 편집기 — 기본정보 → 성취기준 선택 → 4번 반영비율 → 6번 세부기준 + 실시간 검증 |
| `#semester` | 학기 단위 성취수준(2022 전용) — 진술방식 4종, AI 생성, 금지어 검사 |
| `#ai` | AI 추천(6번 자동 반영) · 외부 AI용 프롬프트 · 보완 프롬프트 |
| `#preview` | 3~6번 표 미리보기, 표 단위 복사(한글 붙여넣기), xlsx 다운로드 |

작성 내용은 브라우저(localStorage)에 자동 저장되고, JSON 파일로 내보내 동료와 공유하거나
다른 PC에서 이어 쓸 수 있다.

## AI 기능 켜기 (관리자 1회, 무료)

[docs/GAS-프록시-설치.md](docs/GAS-프록시-설치.md) 참고. Gemini API 키는 Apps Script
스크립트 속성에만 저장되며 교사·저장소에 노출되지 않는다.

## 데이터

`public/data/`에 2022 개정 24과목·2015 개정 19과목의 성취기준/성취수준이 JSON으로 내장.
원천: 교육부·교육과정평가원 보급 자료(xlsx·hwp·pdf). 추출 스크립트는 `scripts/`
(Python, 재실행 시 원본 폴더 경로 필요).

- 원자료의 인접 수준 병합 셀(예: B·D 빈칸)은 위 수준 진술을 물려받고 `mergedUp`으로 표시.
- 검수 화면(`#review`)에서 과목별 추출 결과를 원본과 대조할 수 있다.

## 개발

```bash
npm install
npm run dev        # http://localhost:5173/eval-plan-builder/
```

배포는 main 푸시 시 GitHub Actions가 자동 수행(Pages). 이 개발 PC에서는 로컬
`vite build`가 네이티브 크래시로 실패하는 문제가 있어 프로덕션 빌드는 CI에서만 한다.
